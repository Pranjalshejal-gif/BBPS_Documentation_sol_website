The BillDesk API is organized around REST. Key aspects of the API are:


  1. Uses predictable, resource-oriented URLs
  2. Uses HTTP response codes to indicate API errors
  3. Uses built-in HTTP features, like HTTP authentication and HTTP verbs, which are understood by off-the-shelf HTTP clients
  4. Supports cross-origin resource sharing, allowing the client to interact securely with our API from a client-side web application
  5. JSON is returned by all API responses, including errors, although our API libraries convert responses to appropriate language-specific objects

BBPS CORE Features

With each request, you will need to set the following request headers:

  • Content-Type: Request Content-Type to take the values application/json
  • Accept: Accept Response Content-Type to take the values application/json
  • BD-Traceid: Traceid is used for idempotency and the request with the same Traceid within the day will be rejected. Traceid can be alphanumeric without any spaces or special characters and should be a maximum size of 30.
  • BD-Timestamp: Transaction timestamp in the yyyymmddhh24miss format
  • Authorization: This will be generated by the client as explained in the steps below and validated by BillDesk prior to processing the request. Format of the header is HMACSignature clientid:hmacvalue

  2. Uses HTTP response codes to indicate API errors
  3. Uses built-in HTTP features, like HTTP authentication and HTTP verbs, which are understood by off-the-shelf HTTP clients
  4. Supports cross-origin resource sharing, allowing the client to interact securely with our API from a client-side web application
  5. JSON is returned by all API responses, including errors, although our API libraries convert responses to appropriate language-specific objects
